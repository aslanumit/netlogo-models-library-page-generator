Emergent Shapes - Ellipse

Run on NetLogoWeb

WHAT IS IT?

This model displays a basic conic section: the ellipse. The ancient Greeks called circles, ellipses, hyperbolas and parabolas conic sections because they can be generated geometrically by taking different cross sections of a cone. Later the conic sections were described algebraically as quadratic curves. In this model, the ellipse shape is emergent, generated by the behavior of turtles that are trying to satisfy a distance criterion as below. There are two additional models that demonstrate the use of an agent-based approach to exploring conic sections: Emergent Shapes - Hyperbola, and Emergent Shapes - Parabola.

An ellipse is the set of points such that the sum of the distances to two fixed points (the foci) is constant. The constant is represented in this model by the variable CONIC-CONSTANT.

This definition can be used to draw (or imagine drawing) the ellipse. If you start with a string loosely looped around two nails each representing a focus and then pull the string tight with a pencil point and move the pencil point around the foci, you will draw an ellipse. The length of the string corresponds to the CONIC-CONSTANT.

Here's an equivalent description that may be easier to understand. Pick any two points in the plane and call them the foci of the ellipse. Choose a number that will be referred to as the "conic constant." Now pick any point in the plane and draw the 2 line segments between that point and each of the foci. Measure the length of each of these 2 line segments and add them. If the number is equal to the conic constant then the point you chose is on the ellipse. If not, then it isn't. The shape made by all the points for which their calculation is equal to the conic constant is the ellipse.

HOW IT WORKS

In this model the turtles use feedback to make decisions about how to behave. They set out in random directions and then act on information as to whether or not they are getting closer to where they want to be. If they are getting closer, they continue moving forward in the direction they are going. If they are moving farther away, they set out in a new random direction.

HOW TO USE IT

Create an Ellipse

  • Select the number of turtles with the NUM-TURTLES slider.
  • Choose sum of the distances to the foci of the ellipse with the CONIC-CONSTANT slider.
  • Press SETUP. The turtles will appear in random locations
  • Press the GO button. The turtles will take small random steps at each tick.
  • Click a location to set one focus of the ellipse. The turtles will continue to take small random steps at each tick.
  • Click a different location to set the other focus of the ellipse. The turtles will move until they form an ellipse.

Change the Ellipse

  • Adjust the sum of the distances to the foci of the ellipse with the CONIC-CONSTANT slider. The turtles will automatically adjust themselves.
  • You can move either focus by clicking on it and dragging it.

The Distance Turtle

A "distance turtle" is a randomly chosen turtle used to display the conic "distance." The distance turtle can only be visualized after both foci of the ellipse have been specified. The distance turtle has red line segments connecting it to each focus. Each segment is labeled with its length. The sum of the lengths equals the conic distance set by the slider.

The Monitor

The monitor labeled "conic distance of distance turtle" shows and explains the conic distance. An example would "63.1 = 22.9 + 40.2 = sum of distances to foci."

New Distance Turtle Button

If the ellipse has been specified a new distance turtle is activated. If a distance turtle already exists it is replaced. If the ellipse has not been specified, the distance turtle will be activated after the second focus is chosen.

Clear Distance Turtle Button

If a distance turtle exists it will be deactivated and its red line segments will be removed.

Refresh Turtles Button

This button moves the small green turtles to random locations. If there was a distance turtle it persists.

THINGS TO NOTICE

When forming an ellipse, the turtles try to attain a combined distance of CONIC-CONSTANT from the two foci, again determined by the user's clicks and dragging. If the foci are too far apart, there be no way to satisfy this condition. What do the turtles do then?

THINGS TO TRY

You may be able to get a better feeling for the turtles' behavior if only a few turtles are alive at a time. Try setting num-turtles to a small value (like 16 or 1) and watching the turtles. You can also slow down (or speed up) the movement of the turtles by either using the speed slider on the interfacetool bar, or by clicking on "Settings" and changing the frame rate.

How can you use this model to produce a circle?

Create a distance turtle. Calculate the difference between the labeled lengths of its 2 red line segments. Now move one of the focuses. Recalculate the sum of the lengths of its 2 red lines. Did the calculated number change? Why do you think it stayed the same?

The constant conic-eps is set in the SETUP procedure. It determines how close the turtles have to be to their ideal 'distance' Change its value --- what do you see? Why? Be sure to try some integer values.

EXTENDING THE MODEL

Add a switch which allows the ellipse to wrap the screen instead of stopping at the screen edge.

Add a button that moves one focus by a fixed distance in a random direction.

Add a switch that makes the distance turtle or the segments that extend from it change color depending on whether the conic distance has increased, decreased or stayed the same after its most recent step.

NETLOGO FEATURES

The mouse primitives are used for handling interaction with the user. Link commands are used to display distances from the distance turtle. The ifelse-value primitive is used to set condition dependent values. The "carefully" primitive is used to avoid an error when testing if ticks have been started.
